Critical workarounds
After every Docker restart, reinstall packages in Airflow:
powershelldocker exec -u 50000 mlops-airflow-scheduler python -m pip install scikit-learn pandera boto3 fastparquet mlflow-skinny==2.17.2 scipy --quiet
docker exec -u 50000 mlops-airflow-webserver python -m pip install scikit-learn pandera boto3 fastparquet mlflow-skinny==2.17.2 scipy --quiet
Airflow train task env vars — must be set at top of task_train() before any mlflow import:
pythonos.environ['MLFLOW_S3_ENDPOINT_URL'] = 'http://minio:9000'
os.environ['AWS_ACCESS_KEY_ID']      = 'minioadmin'
os.environ['AWS_SECRET_ACCESS_KEY']  = 'minioadmin'
MLflow version — Airflow must use mlflow-skinny==2.17.2 to match server v2.13.2. Installing without pin installs v3.x which breaks the API.
Pandera — use element-wise lambdas (Check(lambda x: x > 0, element_wise=True)) not Check.greater_than() for Python 3.13 / pandera 0.31 compatibility.
Parquet — use fastparquet, not pyarrow (pyarrow has no Python 3.13 wheel).
